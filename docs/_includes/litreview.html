Literature Review and Related Work

It has been suggested that providing more user control to help steer the recommendation process with additional input and feedback (Millecamp et al., 2018). Especially the participants with high musical sophistication want more active feedback and control, on the other hand for the vast majority automatic playlist continuation is important. For the continuation, successful metrics are required.

Zamani et al. presented an overview the most successful metrics and models for the ACM RecSys Challenge 2018 (Zamani et al., 2019). They found that for most models increasing the number of tracks increased performance. Surprisingly, the models perform worse when the title because models seem to overfit on the titles appearing in the training set. Randomizing track order in the training set may increase performance, because otherwise adjacent tracks are likely to share similar information. The most successful models ensemble the well-known methods (matrix factorization models, neighborhood-based collaborative filtering models, basic information retrieval techniques, and learning to rank models). Including external source information (i.e. outside of the Spotify API may make the models too complex, reducing performance.

According to Lops et al. specific aspects of a recommendation system can only be implemented effectively with knowledge about the features of the items. For that we need knowledge sources that provide information about the items. These sources may include both structured ones like DBPedia, semi-structured ones like Wikipedia (Lops et al., 2019). Based on these insights, we strive to create a sentiment analysis model (based on additional sources and recommender system model (based on similarity to the initial tracks). Lastly, we want to include additional user control by enabling user inputs into the recommender system. In this way we are encompassing the main strategies from the literature in creating the most apt playlist continuation.
